# FlagOn — Product Brief

## 한 줄 정의
"등산하고 정상에 크루 깃발 꽂기" — GPS 인증 기반 산악 영토 경쟁 앱

## 핵심 메카닉
1. **정상 GPS 인증**: 정상 150m 반경 안에서 20분 체류 → 깃발 꽂기 권한 획득
2. **크루 경쟁**: 깃발은 크루 소유. 다른 크루가 같은 정상을 정복하면 깃발 교체
3. **만료 시스템**: 깃발은 7일 후 자동 만료 → 정상 유지를 위해 반복 등산 필요
4. **리더보드**: 활성 깃발 수 기준 크루 랭킹

## 타겟 유저
- 2030 한국 등산 인구 (주말 산행 꾸준히 하는 사람)
- 친구/동료와 함께 다니는 소규모 그룹 (크루)
- 경쟁심 있는 레저 유저 — Strava KOM 개념에 익숙한 층

## 핵심 화면 (3개)
| 화면 | 역할 |
|------|------|
| 🗺 지도 | 주변 정상 + 크루 깃발 현황 지도 표시. GPS 인증 진행 중일 때 타이머 오버레이 표시 |
| 🏆 랭킹 | 크루별 활성 깃발 수 리더보드 |
| 👤 내 정보 | 등정 기록, 소속 크루, 깃발 수 |

## 온보딩 플로우
앱 실행 → Google 로그인 → 지도 진입 (위치 권한 요청) → 크루 참여/생성

## 기술 스택
- React Native (Expo SDK 54, managed workflow)
- iOS 우선 (TestFlight 배포 중)
- Supabase (Auth, DB, Realtime)
- MapLibre (지도)
- CARTO Positron (지도 스타일)

## 차별점
- Strava: 개인 기록 중심 → FlagOn: **팀 영토 경쟁**
- AllTrails: 루트 정보 중심 → FlagOn: **실시간 정상 쟁탈**
- Pokemon GO: 위치 기반 수집 → FlagOn: **실제 등산으로만 획득 가능** (GPS 20분 인증)

## 수익화 방향 (미정)
- 크루 프리미엄 (커스텀 깃발 디자인, 크루 통계)
- 특별 정상 이벤트 (한시적 고점수 지역)
